Base de datos

La base de datos de MusicaList almacena los datos del sistema como lo son las canciones, las listas y la información de los usuarios independientemente sean administrador o votante. cuenta con seis tablas (admin, cancion, lista, persona, persona_cancion y persona_lista), de las cuales dos de ellas corresponden a tablas intermedias (persona_cancion y persona_lista). A continuación, se podrá encontrar detalladamente como se estructurarán las tablas de la base de datos:

Diccionario de datos

Admin

  • id_admin: Corresponde a la llave primaria que identificara de forma única a cada administrador, este valor se asignara de manera automática, es una variable de tipo bigint, no se permitirán nulos en este espacio.

  • contrasenia: Esta variable almacenara la contraseña de ingreso de cada administrador la cual se encontrará cifrada, será de tipo varchar y se permitirá dejar el espacio nulo, el cual pasara a llenarse con un valor NULL.

  •  nombre: Como lo indica el nombre la variable corresponde al nombre del administrador, será una variable de tipo varchar y se permitirá dejar el espacio nulo, el cual pasará a ßllenarse con un valor NULL.
