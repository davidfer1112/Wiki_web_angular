Base de datos

La base de datos de MusicaList almacena los datos del sistema como lo son las canciones, las listas y la información de los usuarios independientemente sean administrador o votante. cuenta con seis tablas (admin, cancion, lista, persona, persona_cancion y persona_lista), de las cuales dos de ellas corresponden a tablas intermedias (persona_cancion y persona_lista). A continuación, se podrá encontrar detalladamente como se estructurarán las tablas de la base de datos:

Diccionario de datos

Admin

Esta tabla almacena información sobre los administradores del sistema, cada fila de la tabla representa un administrador y cada columna representa un dato sobre el administrador, los datos que se almacenan son id_admin, contrasenia y nombre.

Cancion

Esta tabla almacena información sobre todas las canciones que se encuentran almacenadas en la base de datos, por lo que cada fila corresponde a una canción diferente y las columnas a los datos que se almacenarán que corresponderán a id_cancion, album, artista, duracion y nombre.

Lista

En esta tabla se encontrará la información de las todas listas a las que se encontraran vinculadas las canciones, por lo que en cada fila se encontrara una lista con n canciones y en cada columna los datos de cada una de ellas que corresponden al id_lista y genero.

Persona

En esta tabla se almacenarán los datos de los usuarios de tipo persona que corresponden a los votantes, por lo que no cuentan con las facultades de crear listas o subir canciones dentro del sistema. En cada fila de esta tabla se encontrará la información de un usuario y en las columnas los datos que poseen cada uno de ellos que son id_persona, apellido, contrasenia, correo_electronico y nombre.

Persona_cancion

Esta tabla es intermedia entre las tablas persona y canción ya que estas tienen una relación muchos a muchos, con esta se pretende facilitar las relaciones entre las dos tablas de la manera más ordenada posible con el fin de garantizar que se vinculen de las variables de la mejor manera posible. Esta tabla cuenta con las variables id_persona_cacion, like, id_cacion e id_persona

Persona_lista

Por último, esta tabla es una tabla intermedia que relaciona las tablas persona y lista, ya que presentan una relación muchos a muchos, para lo cual se requieren dos llaves foráneas una de la tabla lista y otra de la tabla persona en adición a la variable id_persona_lista y like.
